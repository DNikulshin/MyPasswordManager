# CLAUDE.md — Контекст проекта MyPasswordManager

## Общее описание
MyPasswordManager — это локальный менеджер паролей для Windows, написанный на PowerShell (5.1+).  
Приложение работает в системном трее, предоставляет глобальную горячую клавишу для автозаполнения логинов и паролей, а также графический интерфейс на WPF для управления записями.  
Хранилище паролей зашифровано с использованием AES-256 (через `ConvertFrom-SecureString` с ключом, полученным из мастер-пароля).  
Конфигурация (настройки горячей клавиши) хранится в отдельном JSON-файле.

## Основные компоненты (файлы)

### Main.ps1
- Точка входа.
- Скрывает консоль (WinAPI).
- Проверяет единственный экземпляр через Mutex.
- Запрашивает мастер-пароль (функция `Show-LoginWindow`).
- Инициализирует объект `PasswordVault`.
- Загружает конфигурацию горячей клавиши (`[PasswordVault]::LoadConfig()`).
- Регистрирует глобальную горячую клавишу через `KeyboardHook`.
- Создаёт иконку в трее с контекстным меню.
- Запускает цикл обработки сообщений Windows (`Application.Run`).

**Ключевые функции:**
- `Invoke-PasswordFill` — отображает окно выбора сервиса и выполняет вставку логина/пароля (с учётом флажка «Только пароль»).
- `Update-Hotkey` — вызывается при изменении настроек из GUI, перерегистрирует горячую клавишу.

### Vault.ps1
- Определяет класс `PasswordVault`.
- Управляет зашифрованным файлом `%USERPROFILE%\.my_pm\data.secure`.
- Методы:
  - `SaveEntry`, `GetEntry`, `ListServices`, `RemoveEntry`.
  - `Validate()` — проверяет корректность мастер-пароля (расшифровывает тестовую запись `__validation__`).
  - Статические методы `LoadConfig` и `SaveConfig` — работа с `config.json`.

### Gui.ps1
- Содержит функции для графического интерфейса:
  - `Show-LoginWindow` — окно входа (Windows Forms) с поддержкой показа пароля и ссылкой «Забыли пароль?».
  - `Show-MainWindow` — главное окно на WPF (список записей, поиск, добавление, удаление, кнопка настроек).
  - `Show-SettingsWindow` — окно настройки горячей клавиши (Windows Forms).

**Особенности:**
- При добавлении записи можно временно показать пароль.
- При удалении выводится диалог подтверждения.
- Поиск фильтрует таблицу в реальном времени.
- Кнопка ⚙ открывает настройки, после сохранения вызывает колбэк `OnHotkeyChanged`.

### HotkeyManager.ps1
- Определяет класс `KeyboardHook` (наследует `NativeWindow`), использующий WinAPI `RegisterHotKey`.
- Предоставляет событие `HotkeyPressed` (тип `Action`).
- Корректно освобождает ресурсы при `Dispose`.

### InputSimulator.ps1
- Содержит класс `InputSimulator` с методом `SendText` (использует WinAPI `SendInput`).
- Функция `Send-SecureString` — преобразует `SecureString` в обычный текст и вызывает `SendText`.
- **Примечание:** В текущей версии для автозаполнения используется `SendKeys` из-за проблем совместимости `SendInput` с некоторыми приложениями. Код `InputSimulator` сохранён для возможных будущих улучшений.

### Install.ps1
- Создаёт VBS-лаунчер (`Launcher.vbs`) для скрытого запуска PowerShell.
- Добавляет запись в `HKCU:\...\Run` для автозагрузки.

### Uninstall.ps1
- Удаляет запись из автозагрузки.
- Предлагает удалить хранилище паролей.
- Удаляет файлы программы (кроме самого скрипта) и планирует самоудаление через cmd.

## Хранилище и конфигурация

- **Хранилище паролей:** `%USERPROFILE%\.my_pm\data.secure`
  - Зашифровано AES-256 с ключом, производным от мастер-пароля (SHA-256).
  - Содержит тестовую запись `__validation__` для проверки мастер-пароля.
- **Конфигурация:** `%USERPROFILE%\.my_pm\config.json`
  - Хранит настройки горячей клавиши:
    ```json
    {
      "HotkeyModifiers": { "Ctrl": true, "Shift": true, "Alt": false },
      "HotkeyKey": "P"
    }
