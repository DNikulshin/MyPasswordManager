# 🔐 MyPasswordManager — Лёгкий менеджер паролей для Windows

[![PowerShell](https://img.shields.io/badge/PowerShell-5.1+-blue.svg)](https://github.com/PowerShell/PowerShell)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

Простой, безопасный и полностью бесплатный менеджер паролей, встроенный в Windows.  
Работает без установки сторонних программ, использует только PowerShell и .NET Framework.

## ✨ Возможности

- **Безопасное хранение** — пароли шифруются алгоритмом AES‑256 с ключом, полученным из мастер‑пароля (SHA‑256). Данные хранятся в защищённом файле внутри профиля пользователя.
- **Глобальная горячая клавиша** `Ctrl+Shift+P` — быстрая вставка логина и пароля (или только пароля) в любое приложение.
- **Удобный интерфейс** на WPF — просмотр, поиск, добавление и удаление записей.
- **Проверка мастер‑пароля** — при неверном вводе доступ не предоставляется.
- **Сброс при утере пароля** — встроенная кнопка «Забыли пароль?» для полного удаления хранилища и создания нового.
- **Фоновая работа** — запускается в системном трее, не занимает место на панели задач.
- **Автозагрузка** — установка в реестр через `Install.ps1` с использованием VBS‑лаунчера (окно консоли полностью скрыто).
- **Чистое удаление** — скрипт `Uninstall.ps1` удаляет программу, запись из автозагрузки и (опционально) файл с паролями.

## 📥 Установка

1. **Скачайте** архив репозитория или клонируйте его:
   ```bash
   git clone https://github.com/yourusername/MyPasswordManager.git
Поместите папку MyPM (или MyPasswordManager) в любое удобное место, например C:\Program Files\MyPM.

Разрешите выполнение скриптов PowerShell (если ещё не сделано):

powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
Запустите установку (добавление в автозагрузку):

powershell
cd "C:\Program Files\MyPM"
.\Install.ps1
Готово! При следующем входе в систему менеджер автоматически запустится в фоне.

Примечание: для ручного запуска без автозагрузки используйте Main.ps1 (консоль будет видна) или создайте ярлык на Launcher.vbs (без консоли).

🚀 Использование
Первый запуск
При первом запуске откроется окно входа, где нужно придумать мастер‑пароль.

Важно: мастер‑пароль не хранится и не может быть восстановлен. Запомните его или сохраните в надёжном месте.

Добавление паролей
Кликните правой кнопкой по значку в трее (🛡️) → «Открыть менеджер».

Нажмите «Добавить», заполните поля «Сервис», «Логин», «Пароль».

Можно временно показать пароль, поставив галочку «Показать пароль».

Автозаполнение
Находясь в любом поле ввода (браузер, приложение), нажмите Ctrl+Shift+P.

В появившемся окне выберите нужный сервис.

По умолчанию вставляется связка Логин → Tab → Пароль.
Установите флажок «Только пароль», если нужно вставить только пароль.

Поиск и удаление
В главном окне есть поле поиска (по названию сервиса или логину).

Для удаления записи выделите её и нажмите «Удалить» (потребуется подтверждение).

🛡️ Безопасность
Мастер‑пароль не сохраняется ни в каком виде. Ключ шифрования генерируется на лету с помощью SHA‑256.

Все пароли хранятся в файле %USERPROFILE%\.my_pm\data.secure, зашифрованном алгоритмом AES‑256 через PowerShell ConvertFrom-SecureString.

Доступ к хранилищу возможен только под вашей учётной записью Windows.

Для дополнительной защиты рекомендуется использовать сложный мастер‑пароль и не передавать файл хранилища на другие компьютеры.

🗑️ Удаление
Запустите скрипт деинсталляции:

powershell
.\Uninstall.ps1
Он удалит программу из автозагрузки, спросит о необходимости стереть файл с паролями и предложит удалить все файлы приложения.

text
MyPM/
├── Main.ps1               # Точка входа, основной цикл, трей, горячие клавиши

❓ Часто задаваемые вопросы
В: Почему автозаполнение не работает в Блокноте?
О: SendKeys может игнорироваться некоторыми примитивными приложениями. В браузерах, системных диалогах и большинстве программ всё работает корректно.

В: Я забыл мастер‑пароль. Что делать?
О: Нажмите «Забыли пароль?» в окне входа. Это удалит зашифрованное хранилище, и вы сможете задать новый мастер‑пароль (старые данные восстановить невозможно).

В: Можно ли перенести пароли на другой компьютер?
О: Да, скопируйте папку %USERPROFILE%\.my_pm на новый ПК. При первом запуске укажите тот же мастер‑пароль, что и на старом устройстве.

В: Безопасно ли хранить пароли в файле?
О: Да, файл зашифрован алгоритмом AES‑256. Без мастер‑пароля расшифровать его практически невозможно.
